# 🐿️ 도토리 브레인 연구소 - 새로운 테스트 추가 가이드

테스트 데이터 관리 구조가 개선됨에 따라, 새로운 테스트를 추가하기 위해서는 아래의 단계를 따라주시면 됩니다.

---

## 1. 개별 질문 데이터 생성
`data/tests/` 폴더 안에 새로운 JSON 파일을 생성합니다. 파일명은 테스트의 ID와 일치해야 합니다.
- **경로**: `data/tests/[테스트_ID].json`
- **구조**:
```json
{
  "id": "[테스트_ID]",
  "questions": [
    {
      "q": "질문 내용",
      "options": [
        { "text": "선택지 1", "scores": { "성향_ID_A": 1 } },
        { "text": "선택지 2", "scores": { "성향_ID_B": 1 } }
      ]
    }
  ]
}
```

## 2. 테스트 목록 등록 (메타데이터)
사용자가 메인 화면에서 테스트를 볼 수 있도록 목록에 추가합니다.
- **파일**: `data/tests_meta.json`
- 기존 배열에 새로운 테스트 정보를 추가합니다:
```json
{
  "id": "[테스트_ID]",
  "title": "테스트 제목",
  "desc": "테스트에 대한 짧은 설명"
}
```

## 3. 결과 페이지 생성 (HTML)
`tests/` 폴더 내의 기존 HTML 파일(예: `office_party.html`)을 복사하여 새 파일을 만듭니다.
- **경로**: `tests/[테스트_ID].html`
- 파일 내부 코드는 동일하게 유지해도 되며, 파일명(`[테스트_ID]`)이 데이터 파일명과 일치하면 자동으로 연동됩니다.

## 4. (선택 사항) 결과 유형 및 이미지 설정
만약 새로운 성향(Personality Type)을 추가했다면, 결과 화면에 표시될 제목과 이미지 URL을 등록해야 합니다.
- **파일**: `data/result_config.json`
- `resultImages`와 `resultDescriptions`에 해당 성향 ID의 데이터를 추가합니다.

---

### 💡 팁
- 모든 파일명과 ID는 **영어 소문자와 언더바(`_`)**를 사용하는 것이 좋습니다.
- 로컬에서 테스트할 때는 반드시 **로컬 웹 서버**(Live Server 등)를 사용하여 실행해 주세요. (보안상의 이유로 JSON 파일을 직접 읽어오지 못할 수 있습니다.)
