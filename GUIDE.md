# 🐿️ 도토리 MBTI 테스트 관리 가이드

이 문서는 `tests.json` 데이터가 어떻게 실행되는지, 그리고 새로운 테스트를 추가하거나 기존 내용을 수정할 때 어떤 부분을 변경해야 하는지 설명합니다.

---

## 1. 데이터 연결 흐름 (quiz.js)

`quiz.js` 파일의 하단부에서 데이터를 불러오고 연결하는 로직이 구현되어 있습니다.

1.  **데이터 로드 (Fetch)**:
    *   `scripts/quiz.js`의 약 184행: `await fetch('../data/tests.json')`를 통해 JSON 파일을 비동기적으로 가져옵니다.
2.  **테스트 매칭**:
    *   현재 접속한 HTML 파일의 이름(예: `office_party.html`)을 인식하여 `tests.json` 내의 동일한 `id`를 가진 데이터를 찾아옵니다.
3.  **결과 매핑**:
    *   사용자가 선택한 답변의 점수를 합산한 후, 가장 높은 점수의 키(예: `MOOD_MAKER`)를 찾습니다.
    *   이 키를 사용하여 `quiz.js` 상단에 정의된 `results` 객체에서 최종 타이틀과 설명을 가져와 화면에 보여줍니다.

---

## 2. 직접 수정 및 추가 방법

### A. 질문 및 선택지 수정 (`data/tests.json`)
*   **파일 위치**: `data/tests.json`
*   질문 내용(`q`), 선택지 텍스트(`text`), 그리고 해당 선택지가 어떤 성향의 점수를 올리는지(`scores`) 직접 수정할 수 있습니다.
*   **주의**: 수정 시 JSON 문법(쉼표, 큰따옴표 등)을 정확히 지켜야 합니다.

### B. 결과 메시지 수정 (`scripts/quiz.js`)
*   **파일 위치**: `scripts/quiz.js` 상단의 `results` 객체
*   각 성향 키(예: `PLANNER`, `HOMEBODY`)에 해당하는 **제목(`title`)**과 **상세 설명(`text`)**이 이곳에 저장되어 있습니다. 결과 화면에 나오는 문구를 바꾸고 싶다면 이 부분을 수정하세요.

### C. 결과 이미지 변경 (`data/tests.json`)
*   **파일 위치**: `data/tests.json`의 `resultImages` 섹션
*   성향 키에 연결된 이미지 URL을 수정할 수 있습니다. (현재는 Picsum의 임시 이미지가 연결되어 있습니다.)

---

## 3. 새로운 테스트를 추가하는 순서

새로운 주제의 테스트를 만들고 싶다면 다음 단계를 따르세요.

1.  **JSON 데이터 추가**:
    *   `data/tests.json`의 `tests` 배열 안에 새로운 객체를 만듭니다. (고유한 `id` 설정 필수)
2.  **결과 정보 등록**:
    *   `scripts/quiz.js`의 `results` 객체에 새로운 결과 키와 설명을 추가합니다.
    *   `data/tests.json`의 `resultImages`에 해당 키에 연결될 이미지 주소를 넣습니다.
3.  **HTML 파일 생성**:
    *   `tests` 폴더 안에 `{id}.html` 파일을 만듭니다. (기존 `office_party.html`을 복사해서 이름만 바꿔도 됩니다.)
    *   이 파일이 생성되면 메인 화면(`index.html`)의 리스트에 자동으로 나타납니다.

---

이제 `tests.json`만 수정하면 로직 변경 없이도 내용을 마음껏 업데이트하실 수 있습니다!
